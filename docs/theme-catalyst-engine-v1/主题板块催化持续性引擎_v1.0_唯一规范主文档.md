## 3. 系统目标

本引擎是重大事件工作流中的**主题板块催化副链**。  
它负责把“龙头/监管/产业节点新闻”转化为可验证、可跟踪、可消费的板块信号。

核心目标：

1. 识别“公司新闻是否升级为板块催化”
2. 映射主主题与关联主题
3. 构建并验证主题资产篮子
4. 追踪催化状态：首波 / 延续 / 衰竭 / 失效
5. 输出可供盘前、盘中、个股审计、过夜体检、统一裁决消费的标准结果

## 4. 模块边界

本模块负责：
- theme_event_router
- catalyst_detector
- theme_mapper
- basket_builder
- basket_validator
- continuation_engine
- trade_adapter
- theme_case_archiver

本模块不负责：
- 宏观主链最终定调
- 单股最终买卖
- 执行层下单
- 新闻抓取底层

## 5. 事件范围

固定事件标签：

```yaml
event_scope:
  - macro
  - sector_theme
```

本引擎只处理：

```yaml
event_scope: sector_theme
```

典型样本：
- NVDA 量子模型 → 量子计算板块
- FDA 审批 → 生物科技子板块
- DoD 合同 → 军工 / 无人机子板块
- 龙头 AI / Robotics / Cybersecurity 新闻 → 小赛道持续催化

## 6. 核心状态机

固定状态：

- `FIRST_IMPULSE`
- `CONTINUATION`
- `EXHAUSTION`
- `DEAD`

状态含义：

### FIRST_IMPULSE
- 新闻刚落地
- 板块首次唤醒
- 龙头与高弹性标的开始响应

### CONTINUATION
- 催化未死于首日
- T+1 / T+2 / T+3 仍存在板块扩散
- 价格、广度、媒体跟进仍在协同

### EXHAUSTION
- 新闻仍被讨论
- 但价格跟随减弱
- 广度下降、分化增大、追高风险上升

### DEAD
- 市场不再为该催化定价
- 板块回归原逻辑
- 无新增交易理由

## 7. 主流程

```text
标准化新闻输入
→ 路由到副链
→ 板块催化识别
→ 主题映射

→ 板块联动验证
→ 持续性状态机
→ 交易适配输出
→ 归档与复盘
```

```json
  "raw_text": "..."

```

## 9. 最小输出契约

```json
{
  "event_scope": "sector_theme",
  "primary_theme": "Quantum Computing",
  "current_state": "CONTINUATION",
  "continuation_probability": 0.74,
  "trade_grade": "B",
  "candidate_audit_pool": ["IONQ", "QBTS", "RGTI"]
}
```

输出信封约束（与附件《Contract Versioning & Compatibility》一致）：
- 业务对象可保持最小字段集；
- 但在统一下发前，必须由输出信封补齐并承载以下字段：
  - `contract_name`
  - `contract_version`
  - `producer_module`
  - `safe_to_consume`
  - `fallback_reason`
  - `error_code`
- 下游兼容判定以输出信封中的契约字段为准。
